工作营完整技术 Pipeline：按步骤解析以下将整个工作营（6天，三阶段）分解为 11 个关键步骤，每步标注：必要条件、主要工作、主要成果。步骤 1：数据集准备与预加载项目内容必要条件① 教学团队预采集的高精度 360° 实景全景数据集（≥50 场景 × ≥3 情境变体 = ≥150 张）；② 每张图像附带元数据（时间、GPS、真实情境标签可选）；③ 存储于 SSD 硬盘，路径结构清晰主要工作加载图像库，生成场景 ID 列表；检查图像完整性；建立基础元数据索引（CSV）；划分训练/验证集（若后续需微调模型）主要成果scene_index.csv（含场景ID、文件路径、基础标签）；数据集准备就绪报告步骤 2：多模态 LLM 感知描述生成（E-Agent）项目内容必要条件① 高性能工作站（GPU ≥24GB 显存 或 云端 API 配额）；② 多模态大模型（Qwen2-VL-72B 本地 或 GPT-4o API key）；③ 结构化提示词（生态–社会–经济三维度）主要工作编写批量调用脚本，逐张全景图输入提示词；调用 LLM 生成每场景的结构化描述文本（JSON 格式）；处理异常响应（重试机制）主要成果perception_descriptions.json：每条包含 384/768 维嵌入向量；对全部向量执行 PCA/t-SNE 降维至 2D/3D；生成散点图，按场景类型着色主要成果特征向量矩阵（.npy 或 .csv）；降维可视化图（features_scatter.png）；嵌入模型缓存（加速后续重算）步骤 4：专家校准与交互式权重调整项目内容必要条件① 城市规划专家（教师）介入；② Gradio 或 Streamlit 交互面板；③
