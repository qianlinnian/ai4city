脚本共有 11 个可配置参数，定义在 [morph_metrics_extractor.py](D:/course/ai4city/morph_metrics_extractor.py:702)。

## 1. `--image-dir`

输入图片目录。

```powershell
--image-dir .\resized_images
```

- 默认值：`./resized_images`
- 支持格式：`.jpg`、`.jpeg`、`.png`、`.bmp`
- 只读取目录当前层，不递归查找子目录。
- 图片会按文件名排序后处理。
- 目录不存在或没有图片时，脚本停止。

示例：

```powershell
--image-dir D:\course\ai4city-data\images
```

注意：数据目录仍应保持只读，脚本只会读取图片，不会修改原图。

## 2. `--output-dir`

指定结果输出目录。

```powershell
--output-dir .\model_comparison\segformer_b5_ade20k
```

- 默认值：`./output_results`
- 目录不存在时自动创建。
- 同名结果可能被覆盖，因此不同模型应使用不同目录。

目录结构：

```text
输出目录/
├─ metrics_results.xlsx 或 metrics_results.csv
├─ segmentation_results/
├─ edge_density_maps/
└─ skyline_boundary_maps/
```

MXNet 环境有 `openpyxl`，输出 `.xlsx`；`ai4city-mas` 没有 Excel 引擎，会自动输出 UTF-8 `.csv`。

## 3. `--model-profile`

选择语义分割模型。

```powershell
--model-profile segformer-b5-ade20k
```

默认值：`cityscapes-legacy`

可选 4 种：

| 配置 | 实际模型 | 环境 | 特点 |
|---|---|---|---|
| `cityscapes-legacy` | `deeplab_resnet101_citys` | `ai4city-mxnet` | 19 类城市街景；没有水体类别 |
| `ade20k-resnest101` | `deeplab_resnest101_ade` | `ai4city-mxnet` | ADE20K 150 类；包含多种水体 |
| `segformer-b2-ade20k` | SegFormer-B2 ADE20K | `ai4city-mas` | 速度与精度折中 |
| `segformer-b5-ade20k` | SegFormer-B5 ADE20K | `ai4city-mas` | 更大模型，理论精度更高 |

不能省略模型配置来运行 B2/B5，否则脚本会默认选择 `cityscapes-legacy`。

## 4. `--allow-model-download`

允许自动下载未缓存的模型权重。

```powershell
--allow-model-download
```

这是布尔开关，不需要在后面写 `true`。

- 默认：关闭。
- 设置后：如果模型不存在，允许联网下载。
- 不设置：只使用本地缓存；未缓存则停止并报错。
- 当前四个模型都已缓存，可以省略。

错误写法：

```powershell
--allow-model-download true
```

正确写法：

```powershell
--allow-model-download
```

## 5. `--device`

设置模型运行设备。

```powershell
--device cpu
```

可选值：

- `auto`：自动检测。
- `cpu`：强制使用 CPU。
- `gpu`：强制使用 GPU。

默认值：`auto`

当前环境情况：

- `ai4city-mas` 中 PyTorch 是 CPU 版本。
- MXNet 也按当前安装情况使用 CPU。
- 设置 `gpu` 但环境没有 GPU 支持时，脚本会报错停止。

目前建议明确写：

```powershell
--device cpu
```

## 6. `--limit`

限制本次处理的图片数量。

```powershell
--limit 1
```

- 默认值：无限制，即处理全部图片。
- 图片按文件名排序，处理前 N 张。
- 适合模型验收和调参。
- 必须大于 0。

例如：

```powershell
--limit 5
```

不写 `--limit`：

```powershell
# 处理 image-dir 中所有图片
```

## 7. `--edge-analysis-width`

边缘密度计算时的统一分析宽度。

```powershell
--edge-analysis-width 1024
```

- 默认值：`1024`
- 如果原图宽度大于该值，会按比例缩小。
- 如果原图宽度小于该值，不会放大。
- 必须大于 0。

作用：

- 减少树叶、铺装纹理等过细边缘。
- 让不同分辨率图片的边缘密度更可比较。
- 数值越小，边缘越粗略。
- 数值越大，保留的细节越多。

建议：

| 取值 | 效果 |
|---:|---|
| `768` | 更强降噪，只保留较大轮廓 |
| `1024` | 当前推荐默认值 |
| `1536` | 保留更多细节 |
| `2048` | 较细密，可能重新引入纹理噪声 |

## 8. `--canny-low`

Canny 边缘检测低阈值。

```powershell
--canny-low 100
```

- 默认值：`100`
- 范围：`0～254`
- 必须小于 `--canny-high`

低阈值越小：

- 检出的弱边缘越多。
- 树叶、地砖、阴影等细碎边缘可能增加。

低阈值越大：

- 边缘更少。
- 部分真实但不明显的轮廓可能消失。

## 9. `--canny-high`

Canny 边缘检测高阈值。

```powershell
--canny-high 200
```

- 默认值：`200`
- 最大值：`255`
- 必须大于 `--canny-low`

当前默认组合：

```powershell
--canny-low 100 --canny-high 200
```

如果结果仍太密，可以尝试：

```powershell
--canny-low 120 --canny-high 220
```

如果边缘太少，可以尝试：

```powershell
--canny-low 70 --canny-high 160
```

一次只应改变一组参数，并用相同样本对比。

## 10. `--edge-min-component-pixels`

过滤小型边缘连通域。

```powershell
--edge-min-component-pixels 20
```

- 默认值：`20`
- 像素数少于该值的独立边缘区域会被删除。
- 设置为 `1` 或更小，相当于不进行小连通域过滤。

数值越大：

- 删除更多零碎纹理。
- 边缘密度下降。
- 也可能错误删除小型设施的真实边缘。

建议范围：

| 取值 | 效果 |
|---:|---|
| `10` | 保留较多小边缘 |
| `20` | 当前默认 |
| `30～50` | 更强过滤 |
| `100` | 可能过度过滤 |

## 11. `--skyline-smoothing-ratio`

天际线横向平滑窗口占图片宽度的比例。

```powershell
--skyline-smoothing-ratio 0.015
```

- 默认值：`0.015`，即宽度的约 1.5%。
- 合法范围：大于 0、小于等于 0.2。
- 最终会换算成奇数宽度的平滑窗口。
- 使用循环窗口处理 360° 全景首尾。

例如图片宽度为 2048：

```text
2048 × 0.015 ≈ 31 像素
```

数值越小：

- 更贴近模型原始天空边界。
- 容易保留锯齿和误识别抖动。

数值越大：

- 天际线更平滑。
- 可能削弱真实建筑轮廓变化。

建议范围：

| 取值 | 效果 |
|---:|---|
| `0.005` | 轻微平滑 |
| `0.015` | 当前默认 |
| `0.025` | 较强平滑 |
| `0.05` | 很强，可能损失真实起伏 |
